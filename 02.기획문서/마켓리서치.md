### 2.5 경쟁 분석 시사점

- 네이버/카카오 위치 공유는 임시 링크 방식으로 지속적 모니터링에 부적합하다.
- Life360은 글로벌 1위이나 앱 설치 필수, 한국어 지원 불완전, 고령자 특화 설계 없음.
- 통신사 서비스는 자사 고객 한정이라 범용성 부재.
- 공공 서비스(행복GPS 등)는 치매 진단자 한정, 하드웨어 기기 필요, 신청 절차 복잡.
- **웹 기반 + 고령자 특화 + 부모님 주도 동의 + 통신사 무관**은 국내 공백 포지션이다.

## 3. 타겟 고객 분석

### 3.1 페르소나

#### 페르소나 A: 위치 확인자 (자녀/가족)

| 항목 | 내용 |
|------|------|
| 연령대 | 30~50대 직장인 (부모와 별거 거주) |
| 상황 | 부모님이 치매 초기 또는 독거 생활 |
| 핵심 니즈 | 출근 중·퇴근 후에도 부모님 위치를 즉시 확인하고 싶음 |
| 페인 포인트 | 전화해도 안 받을 때 불안, 앱 설치 부탁하기 어려움 |
| 사용 기기 | PC + 스마트폰 |
| 기대 효과 | 안심 확보, 부모님과 연결감 향상 |

#### 페르소나 B: 위치 제공자 (부모님)

| 항목 | 내용 |
|------|------|
| 연령대 | 65~80세 |
| 상황 | 스마트폰 사용 가능하나 앱 설치·설정에 어려움 |
| 핵심 니즈 | 자녀가 걱정하지 않도록 위치를 알려주고 싶지만 사생활도 지키고 싶음 |
| 페인 포인트 | 복잡한 앱 설정, 배터리 소모 불안, 개인정보 불안 |
| 사용 기기 | 안드로이드 스마트폰 (갤럭시) 주 사용 |
| 기대 효과 | 자녀와 연결감 향상, 언제든 공유 중단 가능한 자율성 |

### 3.2 고객 세그먼트

| 세그먼트 | 규모 추정 | 특징 | 우선순위 |
|----------|----------|------|----------|
| 치매 초기 부모님을 둔 가족 | 약 50만 가구 | 즉각적 모니터링 필요, 비용 지불 의향 높음 | 1순위 |
| 독거노인 가족 | 약 120만 가구 | 일상 안전 확인 목적, 무료 서비스 선호 | 2순위 |

### 3.3 고객 채택 행동

- 앱스토어 검색보다 **네이버/구글 검색**으로 서비스를 발견하는 비율이 높다.
- 자녀가 설정 후 **URL만 공유**하는 온보딩이 선호된다 — 부모님 설치 허들 제거.
- 무료 체험 후 알림 기능 필요 시 유료 전환하는 Freemium 모델이 적합하다.

## 4. 시니어 UX 인사이트

> 출처: 토스 테크 블로그 — 시니어 사용자가 어려워하는 UX 5가지 (NLM-11)

50대 이상 시니어 사용자가 디지털 서비스에서 반복적으로 겪는 오인지 패턴과 본 서비스 설계 원칙:

| 패턴 | 설명 | 서비스 적용 원칙 |
|------|------|-----------------|
| 클릭 가능 요소 오인지 | 텍스트·화살표를 버튼으로 인식 못 함 | ON/OFF 토글을 명확한 버튼 형태 (border, 배경색, 아이콘) |
| 예시 이미지 혼동 | 예시 이미지를 실제 본인 데이터로 착각 | 예시 문구에 "예시" 명시, 개인화 데이터 구분 표시 |
| 모션 그래픽 오해 | 애니메이션을 무조건 따라 하려 함 | 행동 안내를 명확한 텍스트로 보완 |
| 질문형 문구 혼란 | "공유하시겠습니까?" 주체 불명확 | "위치 공유 시작하기" 지시형 문구 사용 |
| 스크롤 미시도 | 화면 아래 콘텐츠 인지 못 함 | 핵심 기능을 스크롤 없이 첫 화면에 배치 |

> 착수보고서 R-05(부모님 낮은 디지털 리터러시)에 대응하는 UX 설계 근거.

### 5.2 실시간 통신 기술

> 출처: NLM-02 SSE vs WebSocket, NLM-05 Supabase Realtime

| 기술 | 특징 | 본 서비스 적용 |
|------|------|---------------|
| SSE (Server-Sent Events) | 단방향(서버→클라이언트), HTTP 기반, 자동 재연결 내장, Vercel 호환 | M1~M4 기본 적용 |
| WebSocket | 양방향, 별도 프로토콜, Vercel Lambda 미지원 | M5+ 서버 분리 후 적용 검토 |
| Supabase Realtime | PostgreSQL 변경사항 실시간 구독, RLS 연동 | SSE 대안으로 검토 가능 |

### 5.3 캐싱 및 성능

> 출처: NLM-03 Redis Node.js

## 8. 포지셔닝 전략

### 8.1 포지셔닝 맵

```
                고령자/부모 특화
                      |
           [우리 서비스]
                      |
웹 기반 ----+----------+----------+---- 앱 기반
                      |
    [네이버/카카오]   [도와줘][Life360][찾아봐]
                      |
                   범용
```

- X축: 웹 기반 (앱 불필요) vs 앱 기반
- Y축: 고령자/부모 특화 vs 범용

### 8.2 핵심 포지셔닝 메시지

> "앱 없이, 링크 하나로. 부모님이 어디 계신지 바로 확인하세요."

| 대상 | 메시지 |
|------|--------|
| 자녀 (위치 확인자) | "앱 설치 없이 URL만 공유하면 부모님 위치를 실시간으로 확인할 수 있습니다." |
| 부모님 (위치 제공자) | "버튼 하나로 공유 시작, 언제든 끌 수 있습니다. 내 위치는 내가 결정합니다." |

### 8.3 차별화 포인트

| 차별화 요소 | 구체적 구현 | 경쟁 대비 우위 |
|-------------|-------------|---------------|
| 앱 설치 불필요 | 모바일 브라우저 바로 동작 | 앱 기반 모든 경쟁사 대비 |
| 고령자 최적화 UI | 큰 버튼, 지시형 문구, 첫 화면 핵심 기능 배치 | 도와줘 제외 모든 경쟁사 대비 |
| 부모님 주도 동의·철회 | ON/OFF 부모님이 직접 제어, 동의 이력 투명 공개 | 전 경쟁사 대비 차별화 |
| 위치 갱신 주기 커스터마이징 | 10초~수 분 설정 (기본 30초) | 전 경쟁사 미제공 |
| 통신사 무관 범용 | 어떤 통신사·요금제도 이용 가능 | 통신사 3사 서비스 대비 |
| 국내 지도 최적화 | 카카오맵 기반, 한국 주소·장소명 정확도 우수 | Life360 등 글로벌 서비스 대비 |

## 11. 결론 및 시사점

| 항목 | 내용 |
|------|------|
| 시장 매력도 | 높음 — 고령화 가속, 독거노인 증가, 공공 서비스 접근성 한계로 민간 시장 수요 확대 |
| 경쟁 강도 | 중간 — 대형 플랫폼은 고령자 특화 부재, 통신사는 자사 고객 한정, 공공 서비스는 진단자 한정 |
| 진입 장벽 | 낮음 — 웹 기술 스택 성숙, 카카오맵 API 활용 가능 |
| 법령 리스크 | 중간 — 위치정보법·개인정보보호법 준수 설계 필수 (동의 분리, 7일 자동 삭제) |
| 핵심 성공 요인 | 고령자 UX 최적화 (토스 5원칙 적용), 부모님 주도 명시적 동의 체계, 빠른 MVP 출시 |
| 주요 기술 리스크 | Android 백그라운드 OS 정책 제한 (R-02) — 1차 출시는 포그라운드 사용 전제로 설계 |

> 부모님 위치 확인 서비스는 초고령사회 진입, 공공 서비스의 접근성 한계, 기존 앱 서비스의 고령자 특화 공백이 맞물리는 시장을 공략한다. MVP에서 핵심 기능(F-01~F-04)을 빠르게 검증하고, 안전구역·이력 기능(F-05~F-07)을 유료화 레버로 활용하는 Freemium 전략이 최적이다. 법령 준수(위치정보법, 개인정보보호법)는 선택이 아닌 필수 설계 요소이며, 이를 투명하게 공개하는 것 자체가 경쟁 차별화 요소가 된다.
