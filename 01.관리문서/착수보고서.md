## 1. 추진 배경 및 목적

### 1.1 추진 배경

고령의 부모님을 둔 가족들이 부모님의 현재 위치를 실시간으로 파악하기 어려운 문제가 존재한다. 특히 치매나 독거 어르신 등 안전 관리가 필요한 상황에서 가족이 빠르게 위치를 확인할 수 있는 수단이 필요하다.

**기존 방법의 한계:**

| 한계 | 설명 |
|------|------|
| 전화 확인 | 부모님이 전화를 받지 않거나 위치를 정확히 설명하지 못하는 경우 발생 |
| 앱 설치 기반 서비스 | 고령자의 낮은 디지털 리터러시로 설치 및 설정 어려움 |
| 상용 위치 공유 앱 | 가족 특화 기능 부족, 복잡한 UI, 별도 구독료 부담 |
| 하드웨어 GPS 기기 | 별도 단말기 구매 비용, 충전 관리 필요 |

### 1.2 목적

| 구분 | 내용 |
|------|------|
| 주목적 | 가족 구성원이 부모님의 현재 위치를 웹 브라우저에서 실시간으로 확인 |
| 부목적 1 | 부모님 명시적 동의 기반의 투명한 위치 공유 체계 구축 |
| 부목적 2 | 별도 앱 설치 없이 스마트폰 브라우저만으로 이용 가능한 서비스 제공 |

### 1.3 기대 효과

| 대상 | 기대 효과 |
|------|-----------|
| 자녀/가족 | 부모님 위치를 언제 어디서나 즉시 확인, 안심 확보 |
| 부모님 | 스마트폰 브라우저만으로 간단히 위치 공유, 가족과 연결감 향상 |
| 사회적 | 치매·독거 고령자 실종 예방, 가족 안전망 강화 |

## 2. 서비스 개요

### 2.1 서비스 특징

- 별도 앱 설치 불필요: 스마트폰 브라우저 접속만으로 이용
- 부모님 주도 동의: 위치 공유 ON/OFF를 부모님이 직접 제어
- 실시간 지도 표시: 카카오맵 기반 부모님 위치 마커 표시
- 가족 그룹: 초대 코드로 가족 단위 접근 권한 관리

### 2.2 사용자 정의

| 역할 | 명칭 | 설명 | 주요 행동 |
|------|------|------|-----------|
| 위치 제공자 | 부모님 | 스마트폰 브라우저에서 GPS 위치를 공유하는 대상 | 위치 공유 동의, ON/OFF 토글, 갱신 주기 설정 |
| 위치 확인자 | 자녀/가족 | 웹에서 부모님 위치를 조회하는 사용자 | 지도 조회, 가족 그룹 관리, 알림 설정 |

### 2.3 사용자 흐름

**부모님 (위치 제공자)**
```
브라우저 접속 → 회원가입/로그인 → 위치 공유 동의 → 공유 시작
```

**자녀/가족 (위치 확인자)**
```
브라우저 접속 → 로그인 → 가족 그룹 참여 → 지도에서 부모님 위치 확인
```

## 5. 비기능 요구사항

| 항목 | 요구사항 | 비고 |
|------|---------|------|
| 플랫폼 | 웹 (PC + 모바일 브라우저 반응형) | 별도 앱 설치 불필요 |
| 위치 정확도 | GPS 기반, 오차 50m 이내 | 브라우저 Geolocation API (`enableHighAccuracy: true`) |

| 보안 | HTTPS 필수, 위치 데이터 암호화 저장, 인증된 가족만 조회 | 브라우저 GPS는 HTTPS 환경 전용 |
| 개인정보 | 부모님 명시적 동의 후 위치 수집, 동의 철회 즉시 중단 | 위치정보의 보호 및 이용 등에 관한 법률 |
| 성능 | 페이지 로드 3초 이내 | Core Web Vitals 기준 |

## 6. 기술 스택 및 아키텍처

### 6.1 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| Frontend | React + 카카오맵 API | 국내 지도 정밀도 및 한국어 주소 검색 지원 |
| 실시간 통신 | SSE (M1~M4) / WebSocket (M5+) | Vercel 서버리스 환경 호환성 |
| Backend | Node.js (Express) | REST API + 실시간 통신 처리 |
| Database | PostgreSQL (Supabase) | 사용자/그룹/위치이력 영구 저장 |
| Cache | Redis | 실시간 위치 캐싱 (30초 TTL) |
| 인증 | JWT + OAuth2 (카카오/구글) | 소셜 로그인으로 부모님 가입 허들 최소화 |
| 배포 | Vercel + Supabase | 1인 운영 최적화, 서버리스 통합 배포 |
| CI/CD | GitHub Actions | 자동 테스트 및 배포 파이프라인 |

### 6.2 Backend 배치 전략

| 단계 | 방식 | 적용 조건 |
|------|------|-----------|
| M1~M4 | Express API Routes → Vercel 통합 배포 | 현재 기본값 |
| M5+ | Express 서버 분리 → Render/Railway 별도 배포 | WebSocket 필요 또는 Lambda 한계 초과 시 |
