# PRD: 부모님 위치 확인 서비스

## 1. 개요

### 배경
고령의 부모님을 둔 가족들이 부모님의 현재 위치를 실시간으로 파악하기 어려운 문제를 해결한다. 특히 치매, 독거 어르신 등 안전 관리가 필요한 상황에서 가족이 빠르게 위치를 확인할 수 있도록 돕는 웹 기반 서비스다.

### 목표
- 가족 구성원이 부모님의 현재 위치를 웹 브라우저에서 실시간으로 확인
- 부모님 동의 기반의 위치 공유 체계 구축
- 별도 앱 설치 없이 웹으로 접근 가능한 서비스 제공

---

## 2. 사용자

| 역할 | 설명 |
|------|------|
| 부모님 (위치 제공자) | 스마트폰을 통해 위치를 공유하는 대상 |
| 자녀/가족 (위치 확인자) | 웹에서 부모님 위치를 조회하는 사용자 |

---

## 3. 핵심 기능

### 3.1 위치 공유 (부모님 측)
- 부모님 스마트폰 브라우저에서 위치 공유 동의 및 활성화
- 주기적으로 GPS 위치를 서버에 전송 (기본: 30초 간격)
- 공유 ON/OFF 토글 버튼 제공

### 3.2 실시간 위치 확인 (가족 측)
- 로그인 후 지도 화면에서 부모님 현재 위치 마커 표시
- 마지막 위치 업데이트 시각 표시
- 위치 갱신 시 자동 지도 업데이트 (폴링 또는 WebSocket)

### 3.3 가족 그룹 관리
- 초대 링크 또는 코드로 가족 그룹 생성
- 그룹 내 여러 부모님 계정 등록 가능
- 가족 구성원별 조회 권한 관리

### 3.4 알림
- 부모님이 설정한 안전 구역(반경)을 벗어날 경우 가족에게 알림 (이메일 또는 브라우저 푸시)
- 위치 공유가 1시간 이상 끊길 경우 알림

---

## 4. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 플랫폼 | 웹 (PC + 모바일 브라우저 반응형) |
| 위치 정확도 | GPS 기반, 오차 50m 이내 |
| 위치 갱신 주기 | 30초 (설정 변경 가능, 최소 10초) |
| 데이터 보관 | 위치 이력 최대 7일 보관 후 자동 삭제 |
| 보안 | HTTPS 필수, 위치 데이터 암호화 저장, 인증된 가족만 조회 가능 |
| 개인정보 | 부모님 명시적 동의 후 위치 수집, 동의 철회 즉시 중단 |

---

## 5. 기술 스택 (제안)

| 영역 | 기술 |
|------|------|
| Frontend | React + 카카오맵 API 또는 Google Maps API |
| Backend | Node.js (Express) 또는 Python (FastAPI) |
| 실시간 통신 | WebSocket 또는 Server-Sent Events |
| 데이터베이스 | PostgreSQL (사용자/그룹), Redis (실시간 위치 캐싱) |
| 인증 | JWT + OAuth2 (카카오/구글 소셜 로그인) |
| 호스팅 | AWS 또는 Vercel + Supabase |

---

## 6. 사용자 흐름

```
[부모님]
브라우저 접속 → 회원가입/로그인 → 위치 공유 동의 → 공유 시작

[가족]
브라우저 접속 → 로그인 → 가족 그룹 참여 → 지도에서 부모님 위치 확인
```

---

## 7. 화면 목록

| 화면 | 설명 |
|------|------|
| 로그인/회원가입 | 소셜 로그인 또는 이메일 인증 |
| 위치 공유 설정 (부모님) | 공유 ON/OFF, 공유 주기 설정 |
| 위치 지도 (가족) | 실시간 지도 + 부모님 위치 마커 |
| 가족 그룹 관리 | 구성원 초대, 삭제, 권한 설정 |
| 안전 구역 설정 | 지도에서 안전 구역 반경 설정 |
| 알림 설정 | 이탈 알림, 오프라인 알림 ON/OFF |

## 8. MVP 범위 (1차 출시)

- [ ] 부모님 위치 공유 (동의 → 실시간 전송)
- [ ] 가족 지도 화면에서 위치 조회
- [ ] 가족 그룹 생성 및 초대

- 위치 이력 조회
- 다중 부모님 계정 지원
